# История изменений (CHANGELOG)

Все заметные изменения в этом проекте будут фиксироваться в этом файле. Формат основан на [Keep a Changelog](https://keepachangelog.com/en/1.0.0/).

## [2.1.0] - 2026-05-30

### Добавлено
*   **Smart Sync (Инкрементальная синхронизация):** Радикальное повышение эффективности сети. Теперь приложение запрашивает только измененные данные (дельты), что экономит до 90% трафика.
*   **Push Notifications (Web Push API):** Полноценная инфраструктура для уведомлений. Настройка и получение напоминаний о дедлайнах прямо в браузере или PWA.
*   **Глобальный поиск:** Мгновенный поиск по задачам с уникальным визуальным фокусом. Несовпадающие пузырьки на графике становятся прозрачными, а нужные подсвечиваются.
*   **Pull-to-Refresh:** Интуитивный жест обновления списка задач на мобильных устройствах.
*   **Focus Sync:** Автоматическая актуализация данных при возврате пользователя в приложение (вкладку).
*   **Настройка частоты "Пульса":** Возможность выбрать интервал автоматического обновления (от 1 до 30 мин) или переключиться в ручной режим для экономии энергии.
*   **Интернационализация (i18n):** Полная поддержка русского и английского языков в интерфейсе и системных сообщениях.

### Изменено
*   **Рефакторинг модальных окон:** Окна редактирования задач и настроек полностью переработаны. Добавлены вкладки, навигация свайпами и фиксированные заголовки/футеры для удобства на длинных списках.
*   **Service Worker:** Переход на `injectManifest` для более гибкого управления фоновыми процессами и пушами.
*   **Улучшение контрастности:** Оптимизация цветовой схемы для лучшей читаемости в обоих режимах темы.

### Исправлено
*   Критический баг с исчезновением задач при перезагрузке страницы в режиме инкрементальной синхронизации.
*   Проблема с дешифровкой настроек при агрегации данных на бэкенде.
*   Безопасность миграций: добавлены проверки существования таблиц для предотвращения случайной потери данных.
*   Устранена утечка памяти в таймерах "Пульса".

## [2.0.0] - 2026-05-28

### Добавлено
*   **Тёмная тема (Dark Mode):** Полноценная поддержка тёмного оформления с автоматическим переключением на основе системных настроек и возможностью ручного выбора.
*   **PWA (Progressive Web App):** Приложение теперь можно установить на смартфон или десктоп как нативное. Поддержка работы оболочки в офлайне и уведомления о выходе новых версий.
*   **Real-time Sync:** Внедрен механизм "пульса", который каждую минуту пересчитывает приоритеты задач без перезагрузки страницы. Автоматический переход к "Новому дню" в полночь.
*   **Шифрование данных (SSE):** Все личные данные пользователей (заголовки задач, заметки, названия категорий) теперь шифруются в базе данных на стороне сервера (AES-256).
*   **Web Workers:** Вынос алгоритма упаковки пузырьков в отдельный фоновый поток. Это обеспечило 60 FPS при взаимодействии с диаграммой даже на слабых устройствах.
*   **Монохромный дизайн:** Новый минималистичный стиль, вдохновленный интерфейсами Apple, с фокусом на контент и адаптивность.
*   **Новые жесты:** Поддержка pinch-to-zoom (щипок) для масштабирования диаграммы и горизонтальные свайпы для переключения между графиком и списком на мобильных устройствах.

### Изменено
*   **Интерфейс:** Полное удаление лишних кнопок ("Обновить", "Новый день") из основного UI. Эти процессы теперь полностью автоматизированы.
*   **Управление масштабом:** На десктопе управление зумом перенесено в компактный оверлей в углу диаграммы.
*   **Настройки:** Реорганизация вкладок настроек. Добавлена кнопка "Принудительная синхронизация" во вкладку "Данные".
*   **Безопасность:** Перевод всех чувствительных полей в БД в формат TEXT для хранения зашифрованных данных.

### Исправлено
*   Ошибки MIME-типов при регистрации Service Worker в среде Laravel.
*   Конфликты между вертикальной прокруткой и горизонтальными свайпами на мобильных устройствах.
*   Проблема "наползания" пузырьков друг на друга при резком изменении размеров окна.

## [1.1.0] - 2026-05-27

### Добавлено
*   **Новые жесты (Mobile):** Долгое нажатие (400мс) для открытия заметок (peek mode) к задаче на мобильных устройствах, вместо конфликтующих hover-событий.
*   **Редизайн Настроек:** Вкладка управления категориями переведена из формата таблицы в компактный список с отдельным вложенным Bottom Sheet окном для редактирования.

### Изменено
*   **Главный экран:** Радикальный редизайн с фокусом на пузырьковую диаграмму (Bubble Chart). Диаграмма теперь занимает всё свободное пространство.
*   **Добавление задач:** Громоздкая форма заменена на компактную строку с "умным" добавлением. Расширенные настройки перенесены в модальное окно.
*   **Списки и фильтры:** Текстовый список задач скрыт по умолчанию. Фильтры превращены в горизонтальную скроллируемую карусель. Управление зумом перенесено в нижнюю плавающую панель для предотвращения перекрытия графика.
*   **Мобильная адаптивность:** Использование `100dvh` вместо `100vh` для точного соответствия высоте экрана мобильных браузеров. Исправлены проблемы с горизонтальным переполнением на узких экранах.

### Исправлено
*   Баг с "залипающими" тултипами при тапах на мобильных устройствах (переход на Pointer Events + отключение системных callout-меню).
*   `TypeError: toFixed is not a function` при рендеринге значений подкатегорий.

## [1.0.0-mvp] - 2026-05-26

### Добавлено
*   **Движок приоритизации:** Математическая модель расчета веса задач (перенесена из Vue 2 референса и улучшена).
*   **Визуализация:** Интерактивная пузырьковая диаграмма (Bubble Chart) с поддержкой масштабирования.
*   **Google OAuth 2.0:** Система авторизации через социальную сеть для облачного хранения данных.
*   **Изоляция данных:** Полная приватность пользовательских данных (tasks, categories, settings) на уровне бэкенда.
*   **Импорт/Экспорт:** Поддержка импорта JSON-бэкапов из оригинального файла `daily.html`.
*   **CI/CD:** Автоматическое тестирование и деплой на сервер через GitHub Actions и SSH.
*   **Брендинг:** Новый дизайн интерфейса в стиле Apple, включая логотип и favicon.

### Техническое
*   Переход на Laravel 13 и Vue 3 (Composition API).
*   Настройка Docker-окружения для разработки.
*   Реализация 100% покрытия тестами критической бизнес-логики (PHPUnit + Vitest).
*   Внедрение Pinia для управления состоянием фронтенда.
